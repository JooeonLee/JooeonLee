## ❓ Why - 어떻게 해당 키워드에 관심이 생겼을까?
### ⚡ 성능 개선
- **집콕**이라는 자취방 조회 프로젝트를 진행했을 당시 **성능이 부족해서 사용성이 좋지 않았던 경험**이 있습니다.
- 문제의 원인을 분석하고 개선하는 과정에서 개발의 즐거움을 느꼈습니다.
- `원인 분석` -> `개선` -> `개선 검증`으로 각 과정을 진행했습니다.
- [지도 기반 매물 최적화](https://daffodil-innocent-4d0.notion.site/RealEstate-API-1cd13ef8bfdd80c7a5bac5527ce11d57)
- [Bulk Insert 기반 삽입 최적화](https://daffodil-innocent-4d0.notion.site/Bulk-Insert-1d413ef8bfdd80b1b5dce8f73e3b4854?pvs=74)
- [🔗ZipKok Repo](https://github.com/JooeonLee/Zipkok-Server)

### 🧪 테스트
- **마이뷰티**라는 화장품 리뷰 공유 프로젝트를 진행했을 당시 빈번하게 변화하는 **요구사항에 따른 코드의 수정으로 수많은 사이드 이펙트를 경험**했습니다.
- 이때부터 변경에 대해 안전한 코드를 작성하는 것에 관심이 생겼고 이러한 관심은 자연스럽게 테스트에 대한 관심으로 이어졌습니다.
- TDD를 적용해보기도 하고 테스트 커버리지를 설정하여 설정 커버리지를 만족해야만 빌드되도록 만들어 보기도 했습니다.
- 이러한 관심은 인턴십 당시 테스트 기반 명세서 작성 자동화라는 성과로 이어져 실제 현업의 개발 생산성 높이는 경험을 했습니다.
- [🔗MyBeauty Repo](https://github.com/JooeonLee/my-beauty)

### ⚙️ CI/CD
- **그레이픽**이라는 사이드 프로젝트를 현재 운영해보고 있습니다. 시작 당시에는 가장 단순한 구조로 배포했지만 **운영을 진행하며 필요에 의해 개선을 진행하며 CI/CD의 매력**을 느꼈습니다.
- 배치 작업 결과를 실시간으로 확인하기 위해 `Slack`알림을 연동했습니다.
- 서버 상태를 점검하기 위해 모니터링 서버를 도입했습니다.
- 배포 시 발생하는 다운타임을 줄이기 위해 무중단 배포을 도입했습니다.
- [그레이픽](https://graypick.co.kr/)
- [🔗GrayPick Repo](https://github.com/JooeonLee/graydang-backend)

### 🔥 그래서 나는 지금?
- 최근에는 배치 시스템을 설계하며 **데이터 정합성**에 대한 고민을 가지고 있습니다.
- 모놀리식 구조에서 볼륨이 커지며 발생하는 관리의 어려움을 느끼며 **멀티 모듈 아키텍처**에 대한 관심을 가지고 있습니다.
- 네트워크I/O에 따른 성능 저하를 개선할 방법으로 **비동기 작업**에 대한 관심을 가지고 있습니다.
